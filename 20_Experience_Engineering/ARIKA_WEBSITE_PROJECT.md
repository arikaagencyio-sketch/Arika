- **Frameworks & Methodology** — presents Arika's own "how we think" artifacts: the confirmed **Revenue Infrastructure Partner** positioning (`02_Offer/OFFER_OS.md` §1) and the Ideal Ascension Model (Gateway Offer → larger infrastructure builds). **Resolved 2026-07-04**: the owner confirmed "360° Growth Revenue Framework" as Arika's real, adopted methodology name (`02_Offer/OFFER_OS.md` §1, §8) — no longer treated as an illustrative working title. Website copy (`frameworks/page.tsx`, homepage) updated to drop the "working name" hedging.
- **Industry Solutions** — **Flag, needs a reframe before copy is written:** the pasted vision lists generic verticals (Healthcare, Finance, Education, Retail, Hospitality, Manufacturing). Sector (01)'s confirmed real ICP is **B2B SaaS only**, 3-tier (`01_Sector/SECTOR_OS.md`: Tier 1 Series A-C $5M-$50M ARR; Tier 2 Post-Seed-Series A $1M-$10M ARR; Tier 3 multi-location niche verticals). To be reconciled against Offer (02)'s real 11-offer catalog outcomes at copywriting time, not invented fresh.

- **Assessments** — **primary, real offer: the Revenue Infrastructure Audit** (`14_Audits_Diagnostics/AUDITS_DIAGNOSTICS_OS.md` §3) — Arika's actual, offer-engineered Gateway Offer, 7 sub-audits (funnel, sales, CRM, automation, acquisition, team, offer), 7-14 day fixed-fee delivery. ## 6. Example imagery — executed 2026-07-04, completed 2026-07-07

Real content and doctrine lives in `19_Design/DESIGN_OS.md` §10 — not duplicated here. **3 of the ~5-6 scenes in the original brief were generated** via OpenArt's `kling-3-omni` model (text2image, 1k, 16:9 — the cheapest real config, per the department's standing "don't spend credits speculatively" doctrine), spending 40 of 40 available Free-plan credits (30 for the 3 kept images, 10 on one necessary retry — see below).

**The remaining 2 scenes (Insights hub, Solutions hero) were completed 2026-07-07 via KIE.ai's Nano Banana Pro connector** (`19_Design/design-plugin/src/integrations/kie-nano-banana-pro.ts`), once OpenArt's Free-plan credits ran out — a second real vendor in Design's Production Engine, not a replacement for OpenArt (see `19_Design/DESIGN_OS.md` §3/§10 for the full doctrine). Both generated clean on the first attempt (no text-artifact retry needed): 16:9, 1K, 18 credits each against the owner's real KIE.ai account (62 of 80 credits remaining after both).

**All 5 real images, downloaded to `arika-website/public/images/` and wired into real pages** (not just generated and left unused):
1. `hero-executive-lobby.png` — Executive Lobby (Creative Digital Twin room), used on Home's "How We Work" section and Case Studies' "Measure" stage.
2. `strategy-war-room.png` — Strategy War Room, used on Home's "The Problem" section, the Frameworks page, and Case Studies' "Diagnose" stage.
3. `revenue-operations-center.png` — Revenue Operations Center, used on Home's "Our Methodology" section and Case Studies' "Build" stage.
4. `insights-intelligence-pod.png` — Executive intelligence briefing pod with abstract data walls, used as the Insights page header image (required adding an optional `image` prop to the hand-built `PageHeader` component — backward-compatible, other pages using `PageHeader` without an image are unaffected).
5. `solutions-outcome-room.png` — Outcome-mapping strategy room with an abstract branching flow diagram, used as the Solutions page header image (same `PageHeader` image prop).

Every prompt baked in the real Brand Genome tokens (Navy `#0E1B29`, Gold `#D4AF37`, Cream `#F7F5F0`, Blush `#F3C1C6`) and the Anti-Stock Law / Executive Infrastructure aesthetic, explicitly instructing "no illustration or 3D-render look... reads as a real photograph, not AI art" per Design's "no visible AI artifacts" hard quality bar (`19_Design/DESIGN_OS.md` §10).

**One real quality defect caught and fixed**: the first `revenue-operations-center` generation contained a garbled, nonsensical wall-sign text artifact ("Executive Enfgnisture") — a textbook AI-generation failure mode. Per the same "no visible AI artifacts" standard, this was treated as a quality failure, not shipped, and regenerated (10 more credits) with explicit "no text, no signage, no letters or numbers anywhere" instructions — the retry is clean and is what's actually in use.

**Claude Design tested once OpenArt credits ran out, found unsuitable for this use case (2026-07-04).** With all 40 OpenArt credits spent, tried Canva's `generate-design` tool (the real MCP surface for "Claude Design," `13_Tech_Stack/TECHSTACK_OS.md` §3) for the remaining Executive Briefing Room scene, using Arika's real Canva brand kit (`kAHOIta05wE`). **Real finding**: `generate-design` produces composed Canva *template* graphics (decorative sparkles, dot-grids, a sunburst, a photo inset in an arch frame) — on-brand in color, but not a clean full-bleed photographic background the way OpenArt's direct text-to-image output was. Confirmed by actually converting a candidate to a real design and exporting it (`create-design-from-candidate` → `export-design`, real downloadable PNG, not a mockup) before judging the fit. Owner reviewed the result and chose to hold off on further imagery generation rather than force a mismatched tool — no image kept from this attempt. Revisit once OpenArt credits refresh (Free-plan resets are typically monthly, not confirmed exactly).

Still open: a real logo file (Canva's Brand Kit panel has no programmatic write access — UI-only, per `19_Design/DESIGN_OS.md` §10). Additional Case Study/Testimonial imagery was considered and deliberately not generated: Case Studies already reuses the 3 Home-page scenes across its Diagnose/Build/Measure stages, and Testimonials is an intentional honest empty state (real testimonials don't exist yet) — generating stand-in client photography there would misrepresent it as real client evidence, which the site's honesty constraint (§7) rules out.

## 7. Real build — `20_Experience_Engineering/arika-website/`

**Started 2026-07-04.** A real Next.js 16 (App Router, TypeScript) project, scaffolded once Relume/Canva/OpenArt were confirmed live (not blocked, contrary to the earlier stale readings — see `13_Tech_Stack/TECHSTACK_OS.md` §3, §8 for the correction).

- **Tailwind v3** (not v4 — Relume's `@relume_io/relume-tailwind` preset requires it) with the real Brand Genome baked in as design tokens: `navy` `#0E1B29`, `charcoal` `#1C1C1C`, `cream` `#F7F5F0`, `gold` `#D4AF37`, `blush` `#F3C1C6`; headings on Space Grotesk (`next/font/google`), body on Inter. **Flag**: Satoshi and Neue Montreal (the Brand Genome's other named headline fonts) are not free Google Fonts — Space Grotesk stands in for now; using the others needs separate licensing/self-hosting, not silently substituted as identical.
- **All 14 Phase 1 pages built and live** (Navbar/Footer moved into the root layout so every page shares them): Home, Assessments, Solutions, Services, Pricing, Book a Strategy Session, About, Contact, Insights, Case Studies, Frameworks & Methodology, Industry Solutions, Proposal Center, Testimonials. Every route verified returning HTTP 200 from a running dev server, plus a clean `npm run build` and `npm run lint` (0 errors) across the whole site.
- **Industry Solutions built using the recommended (not yet owner-confirmed) reframe**: organized by the real SaaS tier/stage ICP (`01_Sector/SECTOR_OS.md`) rather than generic industry verticals — flagged inline in the page's own source comment and in §8, so it's easy to revert if the owner prefers the generic-vertical framing instead.
- **Testimonials built with no fake quotes**: an honest empty-state page explaining that real testimonials will publish here as engagements complete — same treatment Case Studies already used.
- **5 real on-brand images generated and wired in** (3 via OpenArt, 2 via KIE.ai Nano Banana Pro) — see §6.
- **Honesty constraint applied throughout**: no fabricated client names, testimonials, or performance numbers anywhere on the site. Case Studies presents the real Problem→Strategy→Implementation→Results→ROI structure rather than invented client stories; Insights uses the real 7 Content pillars with "coming soon" placeholders; Pricing shows real tier names (Discovery/Growth/Scale/Enterprise) with "Custom" pricing rather than invented dollar figures (no real published price list exists in this repo); "stats" sections use real structural facts (7 sub-audits, 7-14 day delivery, 3 ICP tiers) instead of invented KPIs.
- Two Relume components (`PageHeader`, `CardGrid`) were hand-built rather than fetched from Relume, since the closest Relume equivalents (Page Header 3's search+breadcrumb bar; Comparison 3's competitor feature-matrix) didn't actually fit a solutions/services list — simpler, purpose-built components were the right call, not a forced reuse.
- One real bug caught and fixed in Relume's own fetched code: `Portfolio23`'s original version called `useRef`/`useScroll`/`useTransform` inside a `.map()` callback (a rules-of-hooks violation) — refactored into a proper `ProjectCard` subcomponent so hooks are called at each component's own top level, not conditionally inside a loop.
- **Known limitation not fixed this pass**: `Layout640` (used on Home and Frameworks) still ships Relume's default placeholder YouTube video behind its "play" button — the static thumbnail is now real, but clicking it opens an irrelevant demo video since no real Arika video exists yet. Flagged rather than silently left, since it reads as broken if a visitor actually clicks play.
- **Real logo wired in (2026-07-07)**: exported the owner-confirmed logo (`19_Design/DESIGN_OS.md` §10, Canva design `DAHOWKNcWuw`, "Commanding Logo Featuring Integrated Arrow 'A'") as a real PNG, then used `sharp` (already a project dependency) to precisely crop it into two variants — an icon-only mark (Navbar, favicon/`app/icon.png`) and the full icon+wordmark+tagline lockup (Footer) — since the source design's navy background (`#0E1B29`) is an exact match for the site's own navy scheme background, both crops sit seamlessly with no visible edge. `PageHeader`'s image slot was not touched; this only affects `Navbar24.tsx` and `Footer16.tsx`.
- **Deployed to production (2026-07-07)**: live at `https://arika-agency-website.vercel.app` via Vercel CLI (owner-supplied API token, used only for the deploy, never written into any repo file). All 14 Phase 1 routes verified HTTP 200 on the real deployed URL, not just local dev. `arikaagency.com` added to the Vercel project — one owner action remains: add DNS record `A arikaagency.com 76.76.21.21` at Zoho Domains (the confirmed registrar) to bring the custom domain and auto-provisioned SSL live. Full detail: `00_Agency_Governance/GO_LIVE_CHECKLIST.md` item 30.
