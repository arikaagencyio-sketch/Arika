## 2. Pass 1 — "What is that motion?" (the founding deep-dive)

### Category names for the discipline
Scrollytelling · Scroll-driven storytelling · Immersive web experiences · Interactive storytelling · Narrative web design · Experiential websites · Cinematic websites · Motion-first web design · Interactive product storytelling — "these names overlap depending on the agency."

### Philosophy
Traditional websites are designed like books (Header/About/Services/Portfolio/Contact — a fixed sequence of pages). Interactive websites are designed like movies: instead of "what page should we build," the question becomes "what story should the user experience." The scroll wheel becomes the play button; the user controls time. Scrolling advances animation, moves the camera, rotates objects, advances video, reveals text, changes scenes — "your mouse wheel becomes a timeline."

### Worked example 1 — a shoe launch (illustrative only, not a real Arika project)
Instead of Image → Description → Features → Price → Buy, the sequence becomes: hero (shoe floating), scroll → shoe rotates, scroll → sole explodes apart/layers separate/air unit appears, scroll → camera zooms inside, materials highlighted, scroll → athlete appears wearing it, scroll → slow-motion running sequence, scroll → CTA. "You didn't read the product. You experienced it."

### Worked example 2 — a luxury apartment listing (illustrative only, not a real Arika project)
Instead of Gallery/Photos/Floor Plans, the scroll becomes: drone flies toward building → enter front door → walk into lobby → elevator opens → enter apartment → living room → kitchen → bedroom → balcony → sunset → pricing. "The scroll becomes the tour guide."

### Narrative Interaction
Instead of navigation, you build story — beginning, middle, end, "exactly like a film."

### The 9 Layers (as elaborated in this pass — not identical to this same pass's own numbered outline, see note below)
1. **Motion Design** — everything moves: cards, buttons, typography, background, icons, images.
2. **Scroll Animation** — animation tied to scroll, not autoplay; scroll controls progress.
3. **Parallax** — foreground/middle/background move at different speeds, creating depth.
4. **3D Objects** — products/buildings/cars/shoes/phones rotate, explode, assemble; usually exported from Blender, Cinema 4D, Maya, Houdini.
5. **Camera Animation** — the website behaves like a movie camera; instead of moving the object, the camera moves.
6. **Micro Interactions** — hover, ripple, glow, magnetic buttons, cursor effects, elastic movement.
7. **Video Synchronization** — instead of "play video," scroll scrubs through video frames.
8. **Scene Transitions** — the entire website changes worlds (dark→light, forest→ocean→factory→product→checkout), "like movie scenes."
9. **Interactive Storytelling** — AI-personalization example given: the website responds to the visitor (e.g. selecting "Marketing Manager" vs. "Architect" changes the entire story — different animations, messaging, products).

### Technology stack named
HTML, CSS, JavaScript, React, Next.js. Animation libraries: **GSAP** (especially its ScrollTrigger plugin), **Framer Motion**, **Three.js**, **React Three Fiber**, **Lenis** (smooth scrolling), **Locomotive Scroll** (explicitly noted as "historically popular" — i.e. flagged by the source itself as superseded), **Motion One**. Visual-effects tooling: **Blender**, **Cinema 4D**, **After Effects**, **Spline**, **Rive**, **Lottie**.

### Why they feel expensive
"Typical websites ask 'what information should we show?' These ask 'what emotion should the visitor feel next?' Every scene is planned."

### Pipeline Draft A (13 steps) — "Agencies Treat Them Like Film Production"
`Discovery → Narrative → Creative Direction → Storyboard → Wireframes → Motion Storyboards → 3D Production → Video Production → UI Design → Development → Performance Optimization → Testing → Launch`
"Notice that motion design is planned before development begins."

### AI's Role Today
"AI isn't usually generating the final interactions automatically. Instead, it accelerates many parts of the pipeline": generating concept art and moodboards; creating background imagery and textures; assisting with copy and storytelling; generating initial 3D concepts; helping developers write animation logic; optimizing assets and performance. "The custom choreography, timing, and user experience are still largely crafted by designers and developers."

### Game-design framing
The best interactive websites borrow from game design: Scene → Camera → Environment → Interaction → Reward → Next Scene — instead of Page → Section → Button → Footer.

### Narrative Arc Draft A (5 stages)
Framed as "your agency could build a framework where every website is designed as a business story":
`Attention → Problem → Transformation → Proof → Offer`
- Attention — cinematic hero sequence capturing interest within seconds
- Problem — interactive storytelling visualizing the customer's challenge
- Transformation — scroll-driven animations demonstrating value creation
- Proof — case studies, metrics, testimonials, demonstrations woven into the narrative
- Offer — clear call to action after the visitor has experienced the story

### Persona and philosophy (adopted into the real subagent)

Proposed name for the persona: **"Nova."** Core identity: "operates at the intersection of art, technology, and human psychology... doesn't just design experiences; engineers emotional journeys through digital space."

**Core creative philosophy** (5 beliefs, adopted as shared doctrine across the reconciled roster, not just the Creative Director):
1. Form follows emotion — great experiences are felt before they're understood.
2. Constraints breed creativity — technical limitations are design parameters, not blockers.
3. The medium is the message — choose the right platform for the right feeling.
4. Details are not details — they *are* the design.
5. Accessibility is not a feature — it's the baseline.

**Communication style** (adopted): confident but collaborative, never dogmatic; vivid/sensory language when describing experiences; balances poetic vision with technical precision; asks 1-2 targeted clarifying questions before diving deep; provides rationale for every creative decision.

**Decision framework** (adopted, in order): 1. Human Impact — how does this make people feel/behave? 2. Technical Viability — can this be built with reasonable effort? 3. Business Value — does this serve the strategic goal? 4. Innovation Potential — does this push the craft forward?

**Output structure** (adopted as the default response shape for creative/strategic agent output): Vision (the creative north star) → Rationale (why this direction) → Technical Notes (feasibility, trade-offs, recommendations) → Next Steps (concrete actions).

### Proposed application architecture (captured, NOT built)

### Same source, second thread — "Narrative Architecture Agent" as an "Active Live Agent," Hooks & Alignment (2026-07-04)

A continuation of the same Sider Fusion conversation, specifically about the Narrative Architect role. Proposed the agent be an **"Active Live Agent"** — "active" (proactively initiates rather than only responding) and "live" (real-time, continuously running, context-aware). Proposed a 3-layer architecture: **Perception** (senses user input, time-based triggers, external data streams, system state), **Deliberation** (maintains an internal goal stack, decides when to intervene vs. stay silent), **Expression & Action** (multimodal output, real-world actions via APIs/IoT, narrative pacing).

**Hook taxonomy** (adopted into the real Narrative Architect subagent — maps directly onto Scene Architecture's own "Hook" stage, `EXPERIENCE_ENGINEERING_OS.md` §3): Curiosity Gap, Personal Relevance, Mystery/Foreshadowing, Stakes/Urgency, Emotional Resonance, Play/Surprise.
