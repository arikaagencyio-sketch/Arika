## Phase 3 — Backbone Architecture (synthesized)

**Core systems:** AI Discovery & Audit System, Workflow Orchestration System, AI Agent Deployment System, Integration & API System, Monitoring/Observability System, Training & Adoption System.

**Immutable components:** human-in-the-loop checkpoint on any client-facing AI output (no fully autonomous client-facing AI without a review gate); audit-before-automate (never automate an unmapped process); security/data-handling review required before any system touches client data; documented manual-override path for every automated workflow.

**Customization components:** client's existing tech stack, automation depth (single workflow vs. full orchestration), AI model/vendor selection, integration complexity, team AI-literacy and training depth.

## Phase 4 — Client Journey (synthesized, condensed)

Standard 17-stage journey, Day 1-60. Audit stage is the dominant timeline risk — technical discovery of the client's existing stack is the most likely delay source, consistent with the asset/data-access delay pattern already found across offers #1-3.

## Phase 5 — Internal Execution Journey (synthesized)

Discovery (current-state systems audit) → Architecture Design (workflow mapping, tool selection) → Build (agent/automation development) → Integration & Testing (sandboxed validation before production) → Deployment & Training → Monitoring/Optimization (retainer phase).

**Team roster (standardized across offers #5-11 — see offer #5's Phase 5 note):** Client Partner, Strategy Lead, Implementation Lead, Automation Engineer, Integration Specialist, Analytics/Ops Lead, QA Specialist, Account Manager.

## Phase 8 — Communication Architecture (synthesized)

Weekly technical syncs; a dedicated alert channel for automation failures (distinct need vs. other offers — production automation failures need faster escalation than a standard weekly cadence allows).

## Phase 9 — Client Constraint System (synthesized)

Highest-risk archetypes: **Doubter** (skepticism about AI reliability stalls adoption) and **Unrealistic Client** (expects full autonomy/zero oversight immediately, in tension with the human-in-the-loop immutable). Prevention: lead with the audit's quantified opportunity data, not theory; set the human-in-the-loop requirement explicitly in the SOW as non-negotiable, not negotiable-away under pressure.

## Phase 10 — Quality Control System (synthesized)

Sandbox-before-production gate; security/data-handling review gate; failure-rate benchmark gate (e.g., under 2% automation failure rate required before go-live).
