# BOIS - Brand Operating Intelligence System

BOIS is an execution-grade cognitive branding infrastructure for multi-agent brand intelligence, perception engineering, retrieval-gated execution, and operational orchestration.

## Operating Thesis

Branding is treated as a full-stack operating domain:

- identity embodiment
- perception engineering
- symbolic communication
- emotional architecture
- behavioral influence
- trust signaling
- narrative synchronization
- market cognition
- operational continuity
- governance and adaptation

## Core Design Principles

- Preserve strategic nuance at paragraph level.
- Keep repeated intelligence when it carries reinforcement logic.
- Separate stable identity memory from dynamic execution memory.
- Route tasks through specialist agents with governance gates.
- Grade every output across strategic and operational dimensions.
- Support local + cloud + MCP-aware runtime profiles.

## Architecture Layers

- `core`: engines and runtime components.
- `agents`: 20-agent registry with orchestration contracts.
- `memory`: long-term, semantic, and continuity memory logic.
- `knowledge`: source corpus, indexes, and graph artifacts.
- `workflows`: stage-based execution pipelines.
- `governance`, `validation`, `quality_assurance`: policy and control.
- `grading`, `scoring`: multidimensional audit and scoring.
- `orchestration`: state machine and delegation policy.
- `api`, `ui`: delivery interfaces.
- `documentation`: intelligence maps and implementation guides.

## Runtime Profiles

BOIS is designed for:

- OpenAI Codex local execution
- Claude Code local execution
- cloud microservice execution
- MCP-enabled tool environments

## Execution Entry Points

- `executions/bootstrap_bois.py`: full ingestion, memory, graph, and retrieval bootstrap.
- `core/ingestion/pipeline.py`: recursive ingestion and paragraph preservation.
- `core/retrieval/gated_retriever.py`: retrieval-before-generation enforcement.
- `core/orchestration/engine.py`: stage progression, trigger routing, and agent activation.
- `core/grading/engine.py`: 15-dimension brand scoring.
- `core/governance/policy.py`: governance checks and policy validation.

## Expected Output Standard

Every BOIS output is expected to be:

- sector-specific
- psychographically aligned
- symbolically coherent
- culturally adapted
- operationally synchronized
- governance-compliant
- evolution-ready

## Retrieval Rule

No strategic output is generated until the orchestrator retrieves:

- sector intelligence
- audience psychology
- cultural context
- client memory
- previous outputs or campaign history
- governance rules

If any required source is missing, BOIS returns a retrieval failure state instead of producing generic branding.
